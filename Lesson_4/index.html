Критерії якісної верстки:

  1. Відповідність макету.

    Купляючи ту чи іншу річ ми керуємось певними критеріями які в даному випадку описує макет сайту. Тож не дивно що результат нашої роботи повинен відповідати очікуванням клієнта які описує сам макет замовлення.

  2. Надійність.

    - при зміні контенту верстка не повинна руйнуватись і максимально зберігати свою спроможність користування для відвідувачів.

  3. Зручність.

    - визначає на скільки легко та інтуїтивно зроуміло відвідувачу користуватись версткою. Аналіз адекватності анімацій.

  4. Швидкість.

    - комплекс заходів з оптимізації верстки мeтою яких є зменшення розміру файлів та коду в цілому а також регулювання послідовності завантаження різних елементів сайту. Відображення верстки повинно відбуватись як омога швидше.

  5. Семантика та доступність

    - дружність або не дружність верстки до користувачів з обмеженими можливостями, пошуковими та іншими програмнми роботами які аналізують сайт а також з іншими спеціалістами які вподальшому будуть працювати з кодом.Високий рівень семантики та достуапності реалізується за допомогою певних html тегів та атрибутів.Теги повинні використовуватись за призначенням.

Правила успішності:

  1. Слухай - слухати в моєму рохумінні означає уважність та зосередженість до інформації.

  2. Сприймай - правильне сприйняття напряму залежить від пункту 1 та закладе теоретичний фундамент знань по отриманій інформації.

  3. Довіряй - один із надважливих пунктів для формування правильної організації командної роботи. Довіра один із наших спільних двинунів на дорозі до спільної мети.

  4. Читай - один із пунктів індивідуальної роботи над отриманням теоретичних знань. І перш за все це читання інфо поданої на уроці в текстовому форматі та заміток у групах для того щоб була синхронність думок у навчальному процесі. А інакше думки як вівці занесуть нас на різні полонини :)

  5. Займайся - іще один пункт індивідуальної дисципліни поступовий процес отримання практичних навичок. Тут як і в пункті 4 краще сформувати режим. Під час практичних занятть ми можемо наочно побачити результат відображення теорктичних знань у браузері.

  6. Проробляй - циклічне та поглиблене засвоєння знань та навиків на практиці P.S "Підвищуємо марку цементу у скілах"

  7. Експерементуй - а що - якщо??? Вчимося вирішувати задачі різними способами в пошуку оптимального рішення. Атакуємо задачу з різних флангів. Вкл. креатив та політ думки.

  8. Запитуй - у себе, Google.com і в останню чергу у тих хто виконує попередні 7 пунктів краще за тебе. Чудово смакує в поєжнанні з пунктоми вище.

Правила та підказки викладацього колективу.

  1. @import
    Повинен бути у самому верху перед усім іншим кодом, інакше не працюватиме.

  2. Підключення локальних файлів
    При підключенні локальних шрифтів через '@font-face' потрібно уважно дивитися на жирність шрифту. Його накреслення 'normal', 'italic', 'oblique', а також найчастіша проблема - format("woff"), format("woff2") потрібно писати обовязково в лапках! Інакше працювати не буде.

  3. Найчастіша проблема: При додаванні картинки
    Не забувайте додавати тегу img, CSS властивість 'width' або 'max-width' зі значенням '100%', це не дозволить картинці зламати вашу верстку якщо її замінити на іншу картинку з великим розміром. P.S. Приклад: 'max-width: 100%';

  4. Зовнішні відступи (margin-и)
    Margin-и задаються від 'попереднього' елемента до 'наступного', від першого до другого, зверху вниз, зліва направо.

  5. Схлопування (margin)
    Відступи 'margin-top' і 'margin-bottom' іноді об`єднуються в один, з розміром, що дорівнює найбільшому з них (або розміру одного, якщо вони рівні). Ця поведінка відома як схлопування зовнішніх відступів (margin collapsing). 'Щоб цього не відбувалося, дивись пункт №4.'

  6. Iмена файлів:
    • Імена файлів та папок лише латинські символи
    • Не використовувати пробіли
    • Ніяких спецсимволів
    • Нижній регістр
  7. Правила написання імені класу та ID
    • "Ім'я довільне", але логічне - "описує цей об'єкт"
    • Тільки "латиниця" (eng)
    • Тільки "маленьки" (прописні) літери
    • Пробіли у імені класу (коли в імені більше одного слова) замінюємо на "-" (дефіс)

Селектори

Типи селекторів:

  1. *{ } - усіх тегів
  2. div{ } - конкретних тегів в даному вирадку div
  3. [atribut] { } - тегів з атрибутом
  4. [atribut = "value"] { } - тегів з атрибутом з конкретним значенням
  5. [atribut ~= "value"] { } - значення атрибуту включає вказане як вираз
  6. [atribut |= "value"] { } - значення атрибуту рівнозначне або починається зі вказаного та дифісу
  7. [atribut ^= "value"] { } - значення атрибуту поинається зі вказаного
  8. [atribut $= "value"] { } - значення атрибуту закінчується зі вказаного
  9. [atribut *= "value"] { } - значення атрибуту містить вказане як частина слова чи виразу
  10. .block { } - класу
  11. #id { } - id селектор

Комбінації:

  1. div.text { } усі вказані теги (div) з вказаним класом (text) порядок має значення
  2. .class.class { } усі теги з вказаними класами одночасно
  3. .class, .class { } усі теги з класами в переліку
  4. div.class1.class2 { } усі теги div в яких присутні class1 та class2 одночасно
  5. div.block, p.text { } усі ( div ) з класом .block та усі ( p ) з класом .text

Вкладеність

  1. .class div { } усі теги div на сторіеці які знаходяться на будь якому рівні в об'єкті з класом .class
  2. .block .text { } усі об'єкти з класом text на будь якому рівні в об'єкті з класом block
  3. .text * { } усі об'єкти в середені об'єктів з класом text
  4. .text > p { } yсі теги (р) першого рівня (безпосередні) в середені об'єктів з класом text
  5. .text ~ p{ } усі теги (р) які знаходяться далі по коду після об'єктів з класом text та мають той самий рівень вкладеності
  6. .text + p { } Тег P який знаходяться беспосередньо далі по коду після об'єктів з класом text та мають той самий рівень вкладеності
